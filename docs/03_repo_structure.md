# Repo Structure

The repo is intentionally structured to support self-learning.

- `config/` central project settings
- `docs/` tutorial path
- `sample_data/` multiple CSV inputs
- `sql/` reusable setup and governance SQL
- `src/` reusable Python transformation logic
- `notebooks/` notebook tasks
- `.github/workflows/` repository checks

The goal is to separate:
- project configuration
- reusable code
- notebook orchestration
- SQL governance steps
